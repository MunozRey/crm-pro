# Phase 03 Plan 03.2: invite-member Edge Function Summary

**One-liner:** invite-member Deno Edge Function uses service role key to call inviteUserByEmail() and write a pending invitations row; TeamManagement.tsx wires handleInvite to fetch the function with JWT auth and mock fallback for demo mode

## What Was Built

Implemented AUTH-07: secure team invitation flow that allows org admins/managers to invite teammates by email.

### Security Design

The core constraint: `supabase.auth.admin.inviteUserByEmail()` requires a service role key. Exposing that key in the browser bundle would be a critical security vulnerability. The solution is a Deno Edge Function that reads the key exclusively from `Deno.env`:

```
Browser (anon key only)
  → POST /functions/v1/invite-member (Authorization: Bearer {accessToken})
    → Edge Function (Deno.env: SERVICE_ROLE_KEY)
      → auth.admin.inviteUserByEmail()  ← service role operation
      → invitations.insert()            ← tracks pending state
    ← { success: true, invitationId }
  ← toast.success + form reset
```

### Edge Function (`supabase/functions/invite-member/index.ts`)

1. **CORS preflight** — `OPTIONS` returns `ok` with inline `corsHeaders` (no `_shared/` dependency)
2. **Input validation** — rejects requests missing `email`, `role`, or `organizationId`
3. **Caller authentication** — anon key + Authorization header client calls `getUser()` to verify session
4. **Org membership check** — admin client queries `organization_members` to confirm caller belongs to the org
5. **Role authorization** — only `admin` and `manager` roles can invite; `sales_rep`/`viewer` get 403
6. **Invite email** — `adminClient.auth.admin.inviteUserByEmail(email, { data: { role, organization_id } })` triggers Supabase magic-link email; `user_metadata` carries role/org for acceptance flow
7. **Invitation row** — inserts into `invitations` with `status: 'pending'`; token auto-generates via SQL default
8. **Response** — `{ success: true, invitationId }` on success, `{ error: string }` with appropriate HTTP status on failure

- Added `import { supabase } from '../lib/supabase'`

## User Setup Required

Before the invite flow works in production, the user must:

1. Set `SUPABASE_SERVICE_ROLE_KEY` as a secret in the Supabase Edge Function environment:
   ```bash
   supabase secrets set SUPABASE_SERVICE_ROLE_KEY=<your-service-role-key>
   ```
2. Deploy the function:
   ```bash
   supabase functions deploy invite-member --no-verify-jwt
   ```
   (`--no-verify-jwt` is correct here because the function manually verifies the caller's JWT using the anon key client.)
