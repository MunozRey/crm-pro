# Phase 05.2 Summary — companiesStore & activitiesStore Migrated

## One-liner
Migrated companiesStore and activitiesStore to Supabase following same pattern as contactsStore.

## What was done
- Removed `persist` from both stores
- Added `fetchCompanies()` and `fetchActivities()` with Supabase + seed fallback
- Optimistic CRUD + fire-and-forget Supabase writes in both stores
- snake_case ↔ camelCase mappers for both
- `useDataInit.ts` extended with both fetch calls

## Result
Companies and activities persist in Supabase. localStorage no longer used for either.
