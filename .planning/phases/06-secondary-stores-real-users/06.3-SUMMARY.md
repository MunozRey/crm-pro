# Phase 06.3 Summary — auditStore & customFieldsStore Migrated

## One-liner
Migrated auditStore and customFieldsStore to Supabase; removed 500-entry localStorage cap; custom field values use upsert.

## What was done
- **auditStore**: removed persist + `LS_KEYS` import, removed MAX_ENTRIES cap for Supabase mode, added `fetchEntries()`, `logAction` now also inserts to `audit_log` table
- **customFieldsStore**: removed persist + `onRehydrateStorage`, added `fetchCustomFields()` fetching `custom_field_definitions` and `custom_field_values` in parallel; `setFieldValue` uses upsert with `onConflict: 'entity_id,field_id'`
- Both added to `useDataInit.ts`

## Result
`localStorage.getItem('crm_audit')` returns null. Audit log is queryable from Supabase dashboard. Custom fields persist correctly.
