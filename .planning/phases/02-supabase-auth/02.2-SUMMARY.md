## What Was Built

**Task 1 ā€” initSupabaseAuth rewrite:**
- Replaced `async function initSupabaseAuth()` (returning `Promise<void>`) with synchronous `function initSupabaseAuth(): (() => void) | undefined`
- Single `supabase.auth.onAuthStateChange` listener ā€” no `getSession()` pre-call
- `setIsLoadingAuth(false)` called as first line inside the callback (every event, including `INITIAL_SESSION`)
- Handles `PASSWORD_RECOVERY` event with `window.location.replace('/reset-password')` before any state update
- Correctly maps `AuthUser`: `organizationId` from `app_metadata.organization_id`, `isActive: true`, `updatedAt` from `updated_at ?? created_at`
- Removed `supabaseSession` from `partialize` (Supabase SDK manages its own localStorage key `sb-<ref>-auth-token`)
- Returns unsubscribe function; `App.tsx` `useEffect` now returns it as cleanup
- Mock mode (no Supabase configured): calls `setIsLoadingAuth(false)` immediately and returns

**Task 2 ā€” Tests:**
- `authStore.test.ts`: 4 tests covering INITIAL_SESSION loading resolution, correct AuthUser shape mapping, SIGNED_OUT clearing user, and stub for plan 2.5
- `Login.test.tsx`: 2 tests covering `signInWithPassword` called on submit with correct args, and error message shown on failure
- Both test files use `vi.hoisted()` for mock factory functions (required because `vi.mock()` is hoisted before `const` declarations)
