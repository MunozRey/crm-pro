## Changes Made

### src/lib/supabase.ts
Added a console.warn block after `isSupabaseConfigured` declaration, guarded by `!isSupabaseConfigured && import.meta.env.DEV`. The warning message contains `[CRM]` and explains how to configure env vars.

### src/pages/Register.tsx
Fixed the Supabase branch in `handleSubmit` to destructure `data` from `signUp()` and check `data.session`:
- `data.session` is truthy → `navigate('/')` (immediate login, email confirmation disabled in Supabase)
- `data.session` is null/undefined → `setSuccess(true)` (show "Check your email" screen)

### tests/lib/supabase.test.ts
Replaced stub tests with real SEC-06 tests using `vi.stubEnv` + `vi.resetModules` + dynamic `await import()` to force module re-evaluation with different env vars.

### tests/auth/Register.test.tsx
Replaced stub tests with 4 real tests using `vi.hoisted()` for mock factories (required for hoisted `vi.mock()` to access the mocks). Tests verify: signUp call args, email verification screen on no session, navigate('/') on immediate session, error display on signUp failure.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] vi.mock factory variable hoisting error**
- **Found during:** Task 2 RED phase
- **Issue:** `vi.mock()` is hoisted to top of file, making `const mockSignUp = vi.fn()` inaccessible — "Cannot access 'mockSignUp' before initialization"
- **Fix:** Used `vi.hoisted(() => ({ mockSignUp: vi.fn(), mockNavigate: vi.fn() }))` pattern which is evaluated at hoist time
- **Files modified:** tests/auth/Register.test.tsx
- **Commit:** f820834

**2. [Rule 1 - Bug] Ambiguous placeholder selector for company name field**
- **Found during:** Task 2 RED phase
- **Issue:** `getByPlaceholderText(/empresa/i)` matched both "Empresas" (org name) and "tu@empresa.com" (email) — threw "Found multiple elements" error
- **Fix:** Used `getAllByPlaceholderText(/empresa/i)[0]` to target first match (company name input)
- **Files modified:** tests/auth/Register.test.tsx
- **Commit:** f820834

**3. [Rule 1 - Bug] Button selector used /registr/i but Spanish UI renders "Crear cuenta"**
- **Found during:** Task 2 RED phase
- **Issue:** Plan's `fillAndSubmit` used `getByRole('button', { name: /registr/i })` but `t.auth.registerButton` renders as "Crear cuenta"
- **Fix:** Updated selector to `/crear cuenta/i`
- **Files modified:** tests/auth/Register.test.tsx
- **Commit:** f820834
