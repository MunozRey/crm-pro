## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Replaced inline `const` mock refs with `vi.hoisted()` factories in both test files**
- **Found during:** Task 2 (TDD RED phase — test run)
- **Issue:** Plan's test code used `const mockResetPassword = vi.fn()` at module level, then referenced it inside `vi.mock()` factory. Vitest hoists `vi.mock()` above `const` declarations, causing "Cannot access before initialization" ReferenceError
- **Fix:** Wrapped mock function declarations in `vi.hoisted(() => ({ mockResetPassword: vi.fn() }))` — same pattern used in Register.test.tsx (established in 02.1)
- **Files modified:** tests/auth/ForgotPassword.test.tsx, tests/auth/ResetPassword.test.tsx
- **Verification:** All 4 tests pass; all 19 suite tests pass
- **Committed in:** `6eadb50` (Task 2 commit)
