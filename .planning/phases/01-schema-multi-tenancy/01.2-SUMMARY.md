# Plan 01.2 — Summary

## One-liner
Created the three organizational foundation tables (organizations, organization_members, invitations) and three JWT claim helper functions (set_claim, get_org_id, get_user_role) that all downstream RLS policies and triggers depend on.

## What was built

- `supabase/migrations/` directory created
- `supabase/migrations/20260331000001_create_org_tables.sql` — Wave 1 migration containing:
  - `public.organizations` table (id, name, domain, logo_url, plan, max_users, settings)
  - `public.organization_members` table (with UNIQUE(organization_id, user_id) constraint and FK CASCADE to organizations and auth.users)
  - `public.invitations` table (with cryptographically secure token via gen_random_bytes(32) and 7-day expiry)
  - `public.set_claim()` function — SECURITY DEFINER, writes to auth.users.raw_app_meta_data
  - `public.get_org_id()` function — reads organization_id from JWT app_metadata (O(1), no subquery)
  - `public.get_user_role()` function — reads user_role from JWT app_metadata
  - RLS enabled on all 3 tables with org-scoped and role-checked policies
  - 4 indexes: idx_org_members_org_id, idx_org_members_user_id, idx_invitations_org_id, idx_invitations_token
  - Helper functions defined BEFORE all CREATE POLICY statements (avoids Pitfall 5)
