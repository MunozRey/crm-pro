<task>
  <name>Create realtimeSubscriptions.ts</name>
  <files>src/lib/realtimeSubscriptions.ts</files>
  <action>
Create `src/lib/realtimeSubscriptions.ts`:

```typescript
import { supabase, isSupabaseConfigured } from './supabase'
import { useContactsStore } from '../store/contactsStore'
import { useCompaniesStore } from '../store/companiesStore'
import { useDealsStore } from '../store/dealsStore'
import { useActivitiesStore } from '../store/activitiesStore'
import { useNotificationsStore } from '../store/notificationsStore'

export function initRealtimeSubscriptions() {
  if (!isSupabaseConfigured || !supabase) return () => {}

  const channel = supabase.channel('db-changes')
    .on('postgres_changes', { event: '*', schema: 'public', table: 'contacts' }, () => {
      useContactsStore.getState().fetchContacts()
    })
    .on('postgres_changes', { event: '*', schema: 'public', table: 'companies' }, () => {
      useCompaniesStore.getState().fetchCompanies()
    })
    .on('postgres_changes', { event: '*', schema: 'public', table: 'deals' }, () => {
      useDealsStore.getState().fetchDeals()
    })
    .on('postgres_changes', { event: '*', schema: 'public', table: 'activities' }, () => {
      useActivitiesStore.getState().fetchActivities()
    })
    .on('postgres_changes', { event: '*', schema: 'public', table: 'notifications' }, () => {
      useNotificationsStore.getState().fetchNotifications()
    })
    .subscribe()

  return () => {
    supabase.removeChannel(channel)
  }
}
```

This is a simple refetch-on-change approach. It's not the most efficient (full refetch vs. merging individual events) but it's reliable and correct. Phase 5 doesn't need optimistic merge with `updated_at` guard — that's an optimization we can add later.

The realtime channel is org-scoped automatically because RLS filters the SELECT in fetchContacts/etc.

   - Calls `fetchContacts()`, `fetchCompanies()`, `fetchDeals()`, `fetchActivities()`, `fetchNotifications()` on mount

Only fetch when the user is authenticated (check `useAuthStore.getState().currentUser` or add a dependency). The simplest approach: fetch inside the `AppRoutes` component which is already inside `ProtectedRoute`, or add a new `DataProvider` wrapper. Choose whichever is simplest.

Actually, the cleanest approach: create a small `useDataInit()` hook called inside `AppRoutes` or a `DataLoader` component that wraps the protected routes. This hook checks `isSupabaseConfigured && currentUser` before fetching.
